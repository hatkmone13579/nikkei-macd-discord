# 日経225 MACD クロス検知 → Discord 通知 ⚡v2 Pro Edition

日経225銘柄の **終値ベース MACD(12,26,9)** ゴールデンクロス／デッドクロスを検出し、

PCの常時起動は不要。GitHub Actions 上で完結し、**完全無料**で動きます。

## v2 の特徴

### 🎯 検知エンジン — ただのクロス検出ではなく「質」を採点
クロスした銘柄に **品質スコア（0〜100点）と S/A/B/C グレード**を自動付与します。

| 配点 | 判定内容 | 狙い |
|---:|---|---|
| +30 | 週足MACDの方向と一致 | 上位足順行はダマシが大幅減 |
| +20 | 0ラインに対する位置が有利 | GC:0上／DC:0下 はトレンド継続型で信頼度高 |
| +20 | 200日移動平均と整合 | 大局トレンドとの順行確認 |
| +20 | 出来高確認（25日平均×1.2倍以上） | 商い急増を伴うクロスは本物率が高い |
| +10 | ヒストグラムが加速方向 | 勢いの確認 |
| −15 | 直近10日に他のクロスあり | 往来（ウィップソー）ゾーンを警告 |

グレード：**S（80点〜）／A（60〜）／B（40〜）／C（〜39）**。
さらに**市場全体の地合い**（全銘柄のMACD強気比率＝ブレッドス）と**日経平均自体のMACD状態**も毎日レポート。

### 🖼 UI — Discordで完結するリッチなデイリーレポート
1. **マーケット概況カード**：日経平均の終値・騰落率・MACD状態、地合いゲージ（▰▰▰▰▰▰▱▱▱▱ 62%）、本日の検知数
2. **銘柄カード**：グレードバッジ（🏆S/🥇A/🥈B/🥉C）、スコア、終値・騰落率、確認タグ（週足順行・0ライン上・出来高1.4倍◎ 等）、チャートリンク
3. **チャート画像**：スコア上位3件は **株価+200日線／MACD の2段チャート画像**を自動生成して添付（ダークテーマ・日本語表示）
4. 検知が多い日も**自動分割送信**で切れない

### 🛠 使いやすさ
- **テストモード**：Actions の Run workflow で `test` にチェック → 株価取得なしで疎通・設定を即確認
- **強制実行**：`force` にチェック → 休場日チェック・当日データ未反映チェックを無視して実行
- **設定はすべて環境変数で上書き可**（コード編集不要・`macd.yml` のコメント参照）
- **自動延命**：GitHub Actions の60日無活動停止を keepalive で自動回避（完全放置OK）
- **メンテ通知**：日経225の定期見直し（4月/10月 第1営業日）の30日前に自動でお知らせ

```
└─ nikkei225.csv                  ← 監視銘柄リスト（code,name）※要メンテ
```

## セットアップ（5分）

1. **Discord の Webhook を作る**
   通知したいチャンネルの「設定 → 連携サービス → ウェブフック → 新しいウェブフック」→ URLをコピー。
2. **GitHub にリポジトリを作る**（Public/Private どちらでも可）
   このフォルダ一式を同じ構成でアップロード（`macd.yml` は `.github/workflows/` の中へ）。
3. **書き込み権限を有効化（重要）**
   Settings → Actions → General → Workflow permissions → **「Read and write permissions」**を選択して保存。
4. **Secrets を登録**
   Settings → Secrets and variables → Actions → New repository secret

   | 名前 | 値 | 必須 |
   |------|----|----|
   | `DISCORD_WEBHOOK_URL` | 手順1のURL | ✅ |
   | `ADMIN_WEBHOOK_URL` | エラー通知用（任意） | — |

5. **疎通テスト**
   Actions タブ → nikkei-macd-discord → **Run workflow** → `test` にチェックして実行。
   Discordに「🧪 セットアップテスト OK」が届けば完了。以降は平日19:00に自動実行。

## 銘柄リスト（重要）

`nikkei225.csv` は**サンプル60銘柄**入りです。日経公式の構成銘柄を参照して**全225銘柄**に更新してください。
- 列は `code,name`（証券コード4桁＋銘柄名）、**UTF-8** で保存。
- 入れ替え時期（年2回）はシステムが30日前にDiscordへ通知します。

## 設定（環境変数で上書き／`macd.yml` に追記）

| 変数 | 既定 | 説明 |
|------|------|------|
| `VOLUME_FILTER_ENABLED` | `true` | 出来高未達のクロスを通知から除外（ハードフィルター） |
| `VOLUME_CONFIRM_RATIO` | `1.2` | 出来高しきい値（25日平均比） |
| `MIN_GRADE` | `C` | 通知する最低グレード（例: `B` でC級を非表示） |
| `CHART_TOP_N` | `3` | チャート画像を添付する上位件数（0で無効） |
| `MIN_HIST_ABS` | `0` | 弱いクロスの除外しきい値 |
| `REQUIRE_TODAY` | `true` | 当日データ未反映なら通知せず警告 |

> 💡 おすすめ運用：まず既定（全グレード通知）で1〜2週間様子を見て、通知が多ければ `MIN_GRADE: "B"` に。
> 出来高フィルターをOFF（`false`）にすると、出来高はスコアの一部としてだけ働き、全クロスがグレード付きで届きます。

## トラブルシューティング

- **通知が来ない**：Actionsのログを確認。Secrets名・手順3の権限を再確認。まず `test` 実行で疎通切り分け。
- **「当日の株価がまだ反映されていません」**：Yahoo側の反映待ち。Run workflow で `force` にチェックして再実行。
- **「価格取得失敗が多発」**：yfinance を最新化（`requirements.txt` の更新で解決することが多い）。
- **チャートの日本語が□になる**：フォントインストールのステップが失敗していないかログ確認（コード表記にフォールバックします）。
